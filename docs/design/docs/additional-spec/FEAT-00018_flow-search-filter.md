# FEAT-00018: フロー検索・絞り込み

- 対応する参照: docs/HEARING.md §13.2 #3（検索/フィルター） / docs/spec/TRACEABILITY_REVERSE.md: BD-02-002005（検索・フィルター） / RS-01-003001

## 概要

フロー件数が増えても、サイドバーやクイックコマンドから目的のフローへ短時間で到達できるようにする。既存の平坦な FlowTreeProvider 一覧に対して、名前検索と軽量な絞り込みを追加する。

## スコープ (FEAT-00018-002000)

### 実装する範囲

- フロー名の部分一致検索
- FlowTreeProvider 上での非破壊フィルタ適用
- 補助条件としてのトリガーあり、サブフローあり、更新日時順の優先表示
- クイックコマンドからの検索・選択・オープン
- 一致なしの空状態表示とフィルタ解除アクション

### スコープ外

- ノード設定値、コメント本文、出力履歴を含む全文検索
- 実行履歴を横断した最近実行フィルタ
- 永続保存されるスマートフォルダやタグ管理
- 階層フォルダ管理の再設計

## 仕様

### §3.1 フィルタ状態と導線 (FEAT-00018-003001)

- フィルタ状態は extensionMain で生成する軽量な FlowFilterState で保持し、FlowTreeProvider と検索コマンドを扱う CommandRegistry が共有する
- FlowFilterState は検索語、補助条件、並び順、現在の条件に紐づく補助判定用のメモリキャッシュを保持し、フロー定義自体は変更しない
- flowrunner.searchFlows コマンドで検索語入力と補助条件の選択を開始できるようにする
- flowrunner.clearFlowFilter コマンドで現在の検索条件と補助判定キャッシュを解除できるようにする
- サイドバーと QuickPick は FlowFilterState を共有し、同じ検索条件を参照する
- フィルタ状態の永続化は初期スコープ外とし、現在の VS Code セッション内メモリのみで保持する

### §3.2 フロー名検索 (FEAT-00018-003002)

- 検索対象は flow name のみとし、部分一致で判定する
- 先頭一致に限定せず、空白トリム後の入力を大小文字差異なしで比較する
- 検索語が空の場合は名前条件を無効として全件表示へ戻す
- TreeView 上の表示件数が少ない場合でも同じ検索規則を適用する

### §3.3 補助条件と並び順 (FEAT-00018-003003)

| 条件 | 判定方法 | 初期対応 |
|---|---|---|
| トリガーあり | 対象フローに type=trigger ノードが存在する | 対応 |
| サブフローあり | 対象フローに type=subFlow ノードが存在する | 対応 |
| 最近更新優先 | FlowSummary.updatedAt 降順で並べる | 対応 |
| 最近実行 | 実行履歴との結合で判定する | 初期スコープ外 |

- FlowSummary は id、name、updatedAt のみを持つため、名前検索と更新日時順の並び替えは summary のみで判定する
- トリガーあり、サブフローありの補助条件が 1 つでも有効な場合は、名前検索で絞り込み、summary の updatedAt による候補順を決定した後の候補フローだけを対象に詳細フローを読み込み、nodes 内の type=trigger、type=subFlow の有無で判定する
- 候補フローの詳細ロードは FlowFilterState に紐づくメモリキャッシュを用いて同一条件内で再利用し、refresh、clear、再検索のいずれかで条件が更新された時点で破棄する
- 全件に対する無条件の詳細ロードは行わない
- 複数条件は AND 条件で適用し、名前検索の結果をさらに絞り込む
- 更新日時順の優先表示はフィルタ条件と併用でき、summary の updatedAt 降順で候補順を決定する

### §3.4 サイドバー表示、空状態、解除 (FEAT-00018-003004)

- フィルタ適用中のサイドバーは一致したフローのみ表示する
- 一致件数が 0 の場合、初期対応では TreeView.message は使わず、FlowTreeProvider が空状態用の疑似アイテムを 1 件返して表示する
- 空状態用の疑似アイテムから実行される解除アクションは flowrunner.clearFlowFilter コマンドに統一し、壊れた一覧に見えないようにする
- フィルタ解除後は FlowTreeProvider の通常一覧へ即座に復帰する

### §3.5 クイックアクセス導線 (FEAT-00018-003005)

- 検索コマンドの条件入力は 2 段階とし、第 1 段階で検索語を入力し、第 2 段階で補助条件と並び順を QuickPick で選択する
- 検索語入力欄と QuickPick は現在の FlowFilterState を初期値として反映し、再検索時に既存条件を引き継げるようにする
- 条件確定後の結果一覧から対象フローを直接 openEditor できるようにする
- QuickPick の detail には更新日時と有効な補助条件を表示して、同名フローがある場合でも見分けやすくする
- サイドバーとクイックアクセスの結果一覧は同じ FlowFilterState に基づいて構築する

## 影響範囲 (FEAT-00018-004000)

| コンポーネント | 変更内容 | 影響度 |
|---|---|---|
| extensionMain / FlowFilterState | FlowTreeProvider と CommandRegistry が共有する軽量フィルタ状態の生成と注入 | 中 |
| FlowTreeProvider / IFlowTreeProvider | 共有フィルタ状態、空状態疑似アイテム、並び順の追加 | 高 |
| CommandRegistry | 2 段階の検索 UI、検索開始、解除コマンド、共有条件の初期値反映 | 中 |
| FlowService / Repository 利用箇所 | 補助条件判定のための候補限定詳細ロードとメモリキャッシュ | 中 |
| package.json / NLS | コマンド、メニュー、表示文言の追加 | 中 |

## テスト方針 (FEAT-00018-005000)

- フロー名の部分一致検索で一覧が正しく絞り込まれることを検証する
- トリガーあり、サブフローあり、更新日時順の補助条件が候補限定の詳細ロードと、条件更新時に破棄されるメモリキャッシュで期待どおり適用されることを検証する
- 一致なしの空状態疑似アイテムと clearFlowFilter による解除導線が破綻しないことを検証する
- クイックアクセス導線で既存条件が初期値に反映され、対象フローを直接開けることを検証する
