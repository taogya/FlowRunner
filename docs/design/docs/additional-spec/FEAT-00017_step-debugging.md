# FEAT-00017: ステップ実行デバッグ

- 対応する RD 参照: REV-014 #11, #12, #13 / RS-03 §3.2, §3.3

## 概要

プレースホルダの debugFlow コマンドを実動させ、フローを 1 ノードずつ停止しながら確認できるようにする。既存の DebugService、MessageBroker、WebView のデバッグ導線を接続し、停止中ノードの可視化と中間結果参照を安全に提供する。

## スコープ (FEAT-00017-002000)

### 実装する範囲

- flowrunner.debugFlow コマンドの実動化と、アクティブエディタまたは選択フローを対象にしたデバッグ開始
- デバッグセッションの開始、一時停止、次ステップ実行、停止
- 現在停止中ノードのハイライトとデバッグモード表示
- 停止中ノードの入力プレビュー、および実行済みノードの入出力・エラーの参照
- 無効ノード、条件分岐の非選択経路、ループを含むフローで破綻しない安全なステップ進行
- サブフローノードを 1 ステップ単位で扱う初期対応

### スコープ外

- 条件付きブレークポイント、ウォッチ式、コールスタック表示
- VS Code の言語デバッガ相当の step into / step out
- サブフロー内部へ入り込む詳細ステップ実行
- 実行履歴全体を横断する高度な比較デバッグ

## 仕様

### §3.1 debugFlow コマンド有効化 (FEAT-00017-003001)

- flowrunner.debugFlow はツリービュー項目またはアクティブエディタから対象 flowId を解決する
- コマンド起点とツールバー起点は、同等の検証と debug 開始ロジックを利用し、開始結果が分岐しないようにする
- 対象フローが特定できない場合は警告のみ表示し、セッションを開始しない
- 同時に有効なデバッグセッションは 1 件までとする

### §3.2 デバッグセッションライフサイクル (FEAT-00017-003002)

- デバッグ開始時は先頭から順に評価し、最初の実行可能ノードで一時停止する
- フロー完了または停止操作でデバッグモードを終了し、通常ツールバーへ戻す
- 先頭に無効ノードが連続する場合でも、最初の実行可能ノードまで安全にスキップする
- セッション中の現在位置は DebugService が一元管理し、UI 側で独自の進行状態を複製しない

### §3.3 ステップ進行と停止中ノード表示 (FEAT-00017-003003)

- Next 1 回で次の実行単位 1 件のみを進め、通常ノードは 1 ノード、loop ノードは 1 反復単位、subFlow ノードは呼び出しノード 1 件として扱う
- 停止中ノードは既存の実行状態表示を流用しつつ、デバッグ中であることをツールバーまたは右パネルでも識別できるようにする
- 一時停止位置の更新時は対象ノードを選択状態にし、ユーザーが現在位置を見失わないようにする
- 次ノードが存在しない場合は完了イベントを発火してデバッグモードを終了する

### §3.4 中間結果参照 (FEAT-00017-003004)

- 実行済みノードは DebugService の intermediateResults を用いて、入力、出力、エラー、所要時間を参照できるようにする
- 停止中ノードは、実行前であっても既知の上流出力から構築できる入力スナップショットを表示対象に含める
- 実行済みノードをクリックした場合は、その時点の記録済みスナップショットを再表示する
- 参照可能な結果が存在しない場合は、未実行またはデータなしの空状態を明示する

### §3.5 停止とクリーンアップ (FEAT-00017-003005)

- 停止ボタンと debug:stop メッセージは同じ終了処理を利用する
- 停止時はセッションフラグ、現在位置、分岐スキップ状態、ループ反復状態を破棄する
- ノードハイライトとデバッグ専用 UI 状態は解除する
- 直前までに取得した中間結果スナップショットは、次回開始または画面更新まで読み取り専用で保持してよい

### §3.6 制御フロー別の安全動作 (FEAT-00017-003006)

- enabled=false のノードでは一時停止せず、次の実行可能ノードへ進む
- condition ノードは選択された出力経路のみを追跡し、非選択経路では一時停止しない
- loop ノードは既存の loopState を利用し、反復中でも停止、再開、完了が破綻しないことを優先する
- subFlow ノードは呼び出しノード単位で 1 ステップとして扱い、内部フローの詳細追跡は行わない

## 影響範囲 (FEAT-00017-004000)

| コンポーネント | 変更内容 | 影響度 |
|---|---|---|
| CommandRegistry | debugFlow プレースホルダを実動の開始処理へ置換 | 中 |
| IDebugService / MessageBroker | 開始、停止、イベント転送の接続整理 | 中 |
| DebugService | 一時停止位置管理、中間結果公開、停止処理の補強 | 高 |
| FlowEditorApp / Toolbar / PropertyPanel | 停止中ノード表示、次ステップ、結果参照 UI の追加 | 高 |
| NLS / テスト | デバッグ用文言、UT / IT / E2E の更新 | 中 |

## テスト方針 (FEAT-00017-005000)

- コマンド起点とツールバー起点の両方でデバッグ開始経路が正しく接続されることを検証する
- デバッグ開始、次ステップ、完了、停止のライフサイクルが破綻しないことを検証する
- 一時停止位置の更新に応じてノード表示とデバッグモード表示が更新されることを検証する
- 停止中ノードの入力プレビューと実行済みノードの中間結果参照が可能であることを検証する
- 無効ノード、分岐、ループ、サブフローを含むフローでも安全に停止と継続が行えることを検証する
