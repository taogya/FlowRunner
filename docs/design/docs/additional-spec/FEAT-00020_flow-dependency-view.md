# FEAT-00020: フロー依存関係ビュー

- 対応する RD 参照: RD-01 §10 #6 / REV-014 #6（サブフロー活用時の保守性向上）

## 概要

サブフロー参照を横断的に解析し、選択中フローがどのフローを呼び出し、どのフローから参照されているかを一覧で確認できるようにする。保守時の影響範囲把握を目的とし、重量なグラフエディタを追加しない軽量ビューとして提供する。

## スコープ (FEAT-00020-002000)

### 実装する範囲

- 全フローからの subFlow 参照検出
- 選択中フローに対する outgoing dependency と incoming dependency の表示
- 存在しない flowId を参照する subFlow ノードの警告表示
- 一覧またはツリー形式の軽量ビュー
- 依存先、依存元フローへの移動導線

### スコープ外

- React Flow 上での専用依存グラフキャンバス
- 依存関係の自動修復や一括書き換え
- ワークスペース外リポジトリを横断する依存解析
- 実行時にのみ決まる動的依存関係の追跡

## 仕様

### §3.1 依存関係抽出ルール (FEAT-00020-003001)

- 依存関係は type=subFlow ノードの settings.flowId を参照して抽出する
- 呼び出し元フローから参照先 flowId へ 1 本の outgoing dependency を生成する
- 同一参照先を複数ノードが呼ぶ場合は、参照先単位で集約しつつ参照ノード一覧を保持する
- outputNodeId の有無は依存先識別には使わず、補助情報としてのみ扱う

### §3.2 incoming / outgoing dependency 表示 (FEAT-00020-003002)

- 選択中フローについて「呼び出し先」と「呼び出し元」を分けて表示する
- 各項目には flow name、flowId、参照ノード数、必要に応じて nodeLabel 一覧を含める
- 呼び出し元一覧は全フロー走査から逆引きして生成する
- 対象フローに依存関係がない場合は空状態を表示する

### §3.3 警告表示 (FEAT-00020-003003)

- settings.flowId が空、または参照先が存在しない subFlow ノードは warning として別枠表示する
- warning 項目には参照元フローと問題ノードを特定できる情報を含める
- 欠落参照は incoming dependency には含めず、解決不能な参照として扱う
- フロー削除や rename の後は再計算して警告状態を更新する

### §3.4 軽量ビューと移動導線 (FEAT-00020-003004)

- 初期実装は一覧またはツリー形式を基本とし、重量なカスタムグラフ UI を前提にしない
- 依存先、依存元項目を選択すると対象フローを openEditor できるようにする
- 現在開いているフロー内の参照ノードについては、可能なら該当ノード選択へ連携してよい
- サイクルが存在しても解析は継続し、表示上の情報提供にとどめる

### §3.5 再計算タイミングと空状態 (FEAT-00020-003005)

- フロー保存、rename、delete、明示 refresh 時に依存関係を再計算する
- 選択フローが変わった場合も即座に再計算または再利用可能なインデックスから再表示する
- 依存関係なし、警告なしの状態でも空ビューに見えないように説明文を表示する
- 再計算失敗時はビュー全体を壊さず、既知の結果またはエラーメッセージへ退避する

## 影響範囲 (FEAT-00020-004000)

| コンポーネント | 変更内容 | 影響度 |
|---|---|---|
| 依存解析モジュール | subFlow 参照走査、逆引きインデックス、warning 生成の追加 | 高 |
| FlowService / Repository 利用層 | 全フロー読み込みと再計算契機の追加 | 中 |
| 依存関係ビュー UI | 呼び出し先、呼び出し元、警告一覧の追加 | 高 |
| CommandRegistry / NLS | 依存関係ビュー起点や文言追加の可能性 | 低 |

## テスト方針 (FEAT-00020-005000)

- subFlow ノードから依存関係が正しく抽出されることを検証する
- incoming dependency が逆引きで正しく表示されることを検証する
- 参照先欠落時に warning が表示され、通常依存関係と混同しないことを検証する
- 依存関係なしのフローでも空状態表示が破綻しないことを検証する
