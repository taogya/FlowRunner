# FEAT-00016: 最新実行サマリー

- 対応する RD 参照: REV-010 #1, #6（実行結果確認の効率化と見やすさ改善）

## 概要

最新のフロー実行結果を WebView 上で一覧確認できるサマリー表示を追加する。各ノードを個別にクリックして Output を開かなくても、実行順、成否、所要時間、要約結果をまとめて確認できるようにする。

## スコープ (FEAT-00016-002000)

### 実装する範囲

- WebView 右パネル内への最新実行サマリーセクション追加
- 実行順のノード一覧表示（status、duration、出力要約またはエラー要約）
- サマリー項目クリック時のノード選択・ジャンプ
- 失敗ノードの強調表示
- 実用的な範囲のフィルタ（すべて / エラーのみ）
- 既存の実行結果や状態管理データの再利用

### スコープ外

- 複数回実行の履歴保管
- Output 表示そのものを置き換える高機能ログビュー
- ノードタイプごとの完全な詳細レンダラー再実装

## 仕様

### §3.1 最新実行データ収集 (FEAT-00016-003001)

- 実行完了ごとに、最新 1 回分のサマリー用データを生成して保持する
- 各サマリー項目は少なくとも `nodeId`、`label`、`status`、`durationMs`、`summaryText`、`hasError`、`executionOrder` を持つ
- サマリー表示は既存の実行結果キャッシュやノード状態から組み立て、同じ情報を二重保持しすぎない

### §3.2 サマリー UI 表示 (FEAT-00016-003002)

- 右パネルに「最新実行サマリー」セクションまたはタブを追加する
- 一覧は `executionOrder` 昇順で表示し、各行にステータス、所要時間、1 行要約を表示する
- フィルタは「すべて」と「エラーのみ」の 2 種を基本とする
- 最新実行が存在しない場合は空状態メッセージを表示する

### §3.3 要約テキストと失敗強調 (FEAT-00016-003003)

- 成功時の summaryText は既存出力の先頭要約を再利用し、長文は 1 行で切り詰める
- 失敗時はエラーメッセージ先頭を優先表示し、行全体を視認しやすく強調する
- 強調は既存 UI トーンを崩さない範囲で、アイコン、色、太字のいずれかを組み合わせて行う

### §3.4 ノード選択連携 (FEAT-00016-003004)

- サマリー項目をクリックすると対応ノードを選択し、必要に応じてキャンバスを該当位置へ移動する
- ノードが削除済みなどで存在しない場合は一覧表示のみ維持し、ジャンプ処理は安全に無視する
- 選択後は既存の PropertyPanel / Output 表示と競合しない

## 影響範囲 (FEAT-00016-004000)

| コンポーネント | 変更内容 | 影響度 |
|---|---|---|
| FlowEditorApp 相当の状態管理 | 最新実行サマリーの生成と保持 | 中 |
| PropertyPanel / 右パネル UI | サマリー一覧、フィルタ、空状態表示の追加 | 中 |
| ノード選択 / ビューポート制御 | サマリー項目からのジャンプ処理追加 | 中 |
| 実行結果整形処理 | 1 行要約の組み立てと失敗時の強調情報付与 | 低 |

## テスト方針 (FEAT-00016-005000)

- 最新実行データから実行順サマリーが正しく生成されることを検証する
- サマリー一覧が状態、所要時間、要約を描画することを検証する
- エラーのみフィルタ、失敗強調、空状態表示を検証する
- 一覧クリックでノード選択とジャンプが行われることを検証する
