# FEAT-00015: 実行前バリデーション

- 対応する RD 参照: REV-010 #5, #6（設定ミスの早期検出と実行前確認の強化）

## 概要

フロー実行やデバッグ開始の直前に、明らかな設定ミスや参照不整合を検出する。実行後に失敗へ気づくのではなく、開始前に修正または確認できる導線を追加する。

## スコープ (FEAT-00015-002000)

### 実装する範囲

- 実行開始およびデバッグ開始の直前バリデーション
- バリデーションカテゴリの追加（必須設定不足、危険な空欄、未解決サブフロー参照、AI モデル未選択、ファイル操作の基本設定不備、重要分岐の未接続、トリガー設定不足）
- severity を high / medium / low で返す結果一覧
- high が 1 件でもある場合は実行開始を中止し、medium / low のみの場合はユーザー確認で継続可とする
- コマンドごとの個別チェックではなく、再利用可能な Validator Service / Interface に集約する

### スコープ外

- 実行時データを使わないと確定できない動的エラーの完全検出
- バリデーション結果からの自動修正
- WebView 側だけで完結する重複チェック実装

## 仕様

### §3.1 Validator Service インターフェース (FEAT-00015-003001)

- フロー定義と実行コンテキストから問題一覧を返す専用サービスを追加する
- 返却要素は少なくとも `severity`、`message`、`nodeId`、`nodeLabel`、`category` を持つ
- 実行開始コマンドとデバッグ開始コマンドは同じサービスを利用し、ノード実行前の散在チェックを増やさない

### §3.2 バリデーションカテゴリ (FEAT-00015-003002)

| カテゴリ | 例 | 既定 severity |
|---|---|---|
| 必須設定不足 | command ノードの command 未設定、http ノードの URL 未設定 | high |
| 危険な空欄 | 空文字のまま保存された実行対象パス、削除系操作の対象未指定 | medium |
| 未解決サブフロー参照 | subflow ノードの参照先が存在しない | high |
| AI モデル未選択 | aiPrompt ノードで model 未設定 | high |
| ファイル操作の基本設定不備 | read / write / append / delete のモードとパスの整合不備 | high |
| 重要分岐の未接続 | true / false、body / done など重要ポートの片側未接続 | medium |
| トリガー設定不足 | schedule の cron 未設定、event 種別未選択 | medium |

- severity はルールごとに見直し可能だが、初期実装では上表を既定値とする
- low は注意喚起用に予約し、初期実装では medium 相当の補助警告を格下げして利用できるようにする

### §3.3 結果表示と実行ゲート (FEAT-00015-003003)

- バリデーション結果は severity 順に並べた一覧ダイアログまたはメッセージリストで提示する
- high を含む場合は「修正して再実行」を促し、実行とデバッグの開始を許可しない
- medium / low のみの場合は「続行」「キャンセル」を選べる確認を表示する
- 表示メッセージには nodeLabel または nodeId を含め、修正対象を特定できるようにする

### §3.4 コマンド導線統合 (FEAT-00015-003004)

- 実行開始、デバッグ開始、将来の再実行導線は同じバリデーションサービスを呼ぶ
- 実行サービス本体はバリデーション通過後にのみ開始し、途中で再度同種チェックを重複実装しない
- 既存の executor 単位 validate がある場合は、集約可能なものを本サービスへ寄せる

## 影響範囲 (FEAT-00015-004000)

| コンポーネント | 変更内容 | 影響度 |
|---|---|---|
| 実行開始 / デバッグ開始コマンド | バリデーション呼び出しと継続確認の追加 | 中 |
| Validator Service / Interface | バリデーション結果の集約ポイントを新設 | 高 |
| ノード設定メタデータ | 必須項目や重要分岐の判定情報を再利用または補強 | 中 |
| メッセージ表示層 | severity 別の結果一覧表示 | 中 |

## テスト方針 (FEAT-00015-005000)

- Validator Service がカテゴリ別の問題を正しい severity で返すことを検証する
- high を含む場合に実行開始とデバッグ開始がブロックされることを検証する
- medium / low のみの場合に確認ダイアログの選択で継続可否が分岐することを検証する
- 実行導線が共通サービスを利用していることを検証する
