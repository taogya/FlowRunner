# FEAT-00014: スターターテンプレート拡充 + 新規作成導線改善

- 対応する RD 参照: RD-01 §10 #4（定型フローの雛形を提供）
- 関連ヒアリング: REV-010 #4, #13

## 概要

初回利用者が空のキャンバスから考え始めなくても、用途別スターターテンプレートから実用的なフローをすぐ作成できるようにする。既存テンプレート機能を拡張し、新規作成コマンドの導線を「空白」「スターターテンプレート」「最近使ったテンプレート」に整理する。

## スコープ (FEAT-00014-002000)

### 実装する範囲

- スターター向けビルトインテンプレートの拡充（command-runner, file-reader, api-checker, scheduled-cleanup, error-handling-sample）
- テンプレートメタデータの追加（難易度、推奨ユースケース、タグ）
- 新規作成コマンドの初手選択肢として「空白」「スターターテンプレート」「最近使ったテンプレート」を追加
- QuickPick ベースのプレビュー表示（description/detail または確認ステップで概要を提示）
- スターターテンプレート内へのチュートリアル用コメントノード埋め込み

### スコープ外

- WebView ギャラリーや画像サムネイル付きテンプレートブラウザ
- テンプレート評価、ランキング、外部共有
- テンプレートのクラウド同期

## 仕様

### §3.1 スターターテンプレート定義拡張 (FEAT-00014-003001)

- ビルトインテンプレートにスターター系 5 種を追加する
- 各テンプレートは既存のテンプレート定義に加え、`difficulty`、`recommendedUseCase`、`tags` を持つ
- `difficulty` は beginner / intermediate の 2 段階とし、QuickPick 表示で判別できるようにする
- `recommendedUseCase` は 1 文で想定用途を示し、`tags` は複数指定可とする

### §3.2 新規作成導線の分岐 (FEAT-00014-003002)

- 既存の新規フロー作成コマンド開始時に、作成方法を選ぶ QuickPick を最初に表示する
- 「空白」は既存の空フロー作成経路をそのまま利用する
- 「スターターテンプレート」はスターター用テンプレート一覧を表示する
- 「最近使ったテンプレート」は直近 5 件以内のテンプレート履歴を表示し、履歴が空の場合はスターターテンプレート選択へ誘導する

### §3.3 テンプレート選択補助 (FEAT-00014-003003)

- テンプレート一覧 QuickPick の title または placeholder で用途を案内する
- 各候補の description/detail に推奨ユースケース、難易度、タグ、主要ノード構成を表示する
- QuickPick の detail だけで情報が不足する場合は、選択後の確認ステップでテンプレート概要を再表示してからフロー名入力へ進む
- テンプレートからフロー生成する際は既存の ID 再割り当てルールをそのまま使う

### §3.4 チュートリアルコメントノード (FEAT-00014-003004)

- スターターテンプレートには 2 から 4 個のコメントノードを同梱し、開始地点、必須設定、期待結果、次のカスタマイズ候補を案内する
- コメントノードは実行対象に含めず、テンプレート利用者が削除してよい補助情報として扱う
- コメント内容はテンプレート固有の具体例を含め、空欄のまま残さない

## 影響範囲 (FEAT-00014-004000)

| コンポーネント | 変更内容 | 影響度 |
|---|---|---|
| テンプレート定義モジュール | スターターテンプレート追加、メタデータ拡張 | 中 |
| 新規フロー作成コマンド | 初手 QuickPick 分岐、最近使ったテンプレート導線追加 | 中 |
| テンプレート履歴管理 | 最近使ったテンプレートの保存・取得 | 低 |
| NLS / コマンド表示文言 | QuickPick 項目、プレビュー文言追加 | 低 |

## テスト方針 (FEAT-00014-005000)

- スターターテンプレート定義が必要メタデータを持つことを検証する
- 新規作成コマンドの分岐が空白、スターター、最近使ったテンプレートで正しく切り替わることを検証する
- テンプレートプレビュー情報とフロー生成結果が一致することを検証する
- チュートリアルコメントノードがスターターフローに含まれることを検証する
