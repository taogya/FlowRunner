# ST-01: システム試験計画書

## 1. はじめに

本書は RS（要件定義）で定義した全 40 セクションに対するシステム試験の仕様を定義する。V モデルにおける ST（システム試験）は RS に対応する検証工程である。

### テスト手段の分類

| 分類 | ツール | 対象 |
|---|---|---|
| 自動（e2e） | @vscode/test-electron + Mocha | Extension Host 内で実行可能な機能検証 |
| 自動（UI） | vscode-extension-tester + Selenium | サイドバー・コマンドパレット・エディタタブ等の UI 操作検証 |
| 手動 | 開発者による VS Code 操作 | WebView UI 操作、視覚的確認、ユーザーインタラクション |

### テストファイル対応

| テストファイル | 対象 RS | 自動テスト数 |
|---|---|---|
| `src/test/e2e/RS-01.test.ts` | RS-01（UI構成と操作） | 18 |
| `src/test/e2e/RS-02.test.ts` | RS-02（ノード定義） | 14 |
| `src/test/e2e/RS-03.test.ts` | RS-03（実行エンジン） | 12 |
| `src/test/ui/RS-01-ui.test.ts` | RS-01（UI構成と操作）UI | 6 |
| `src/test/ui/RS-03-ui.test.ts` | RS-03（実行エンジン）UI | 1 |

### テスト実行手順

#### 自動テスト（e2e）

```bash
npm run test:integration
```

#### 自動テスト（UI）

```bash
npm run test:ui
```

#### 手動テスト

1. `npm run package` で VSIX をビルド
2. VS Code に拡張機能をインストール
3. 本書の手動テスト項目を上から順に実施
4. 各項目の結果を ✅ / ❌ で記録
